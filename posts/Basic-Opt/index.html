OI 知识梳理 I：基础优化技巧

发表于 2022-12-20 更新于 2023-03-27 分类于 Notes

这里包含了一部分比较吃经验积累的内容，各个算法都有许多变种和应用。值得一题的是，本篇包含的许多内容，例如 CDQ 分治、树分治、整体二分、DSU On Tree 等算法常常都被归到数据结构一类，实际上很不妥，于是将其移出来专门开一篇博客叫做“基础优化技巧”。

1. 二分系列

1.1 基础二分

基础的二分是对单调性的利用，只要有单调性的都可以尝试二分。二分最基础最常见的问法是“最大值最小”和“最小值最大”。

分数规划：当求一个分式的最值时，考虑二分一个答案 midmidmid，转而判断 ≥mid\ge mid≥mid 或 ≤mid\le mid≤mid 是否可行，此时可以将分母移过去，问题大大简化，这个套路被称为“分数规划”。

三分：对于传统的定义域在实数上的凸函数求最值，可以使用三分法，取两个位置比较大小，舍去较劣的部分。

对凸函数的二分：OI 中一般碰到的是离散的凸函数，其差分是单调的。所以求最值只需要使用二分法。

倍增：将带有结合律信息进行二进制拆分，仅用 O(log⁡)O(\log)O(log) 段信息合并出答案。

1.2 wqs 二分

1.2.1 算法介绍

基于凸性并通过二分斜率来得到答案的二分答案，被称作 wqs二分。一般可以用来计算“刚好选 nnn 个的答案最大值”的问题，来对 DP 降维。

使用有一个前提，设 g(i)g(i)g(i) 表示选 iii 个时的最优方案，那么将所有点 (i,g(i))(i, g(i))(i,g(i)) 画出之后所有点必须组成一个凸包（上凸下凸都可以），因此斜率单调递增或递减，可以二分。

发现随着斜率 kkk 的减小，这些直线切的点会越来越靠右。尝试二分一个斜率 kkk，现在问题转换成找到被切的点。设直线的截距 bbb，那么经过点 (i,g(i))(i, g(i))(i,g(i)) 的斜率为 kkk 的直线在 yyy 轴上面的截距为 g(i)−kig(i) - kig(i)−ki，接着就转换成找到最大的截距，考虑这个式子的含义，相当于求选的每个物品的价值都减去 kkk 之后的最优解，这样就没有了选择物品个数的限制可以直接求解。如果当前的 kkk 算出来的最优解所在的 iii 比 nnn 大，kkk 就应该增大，否则减少。

注意：遇到 wqs 二分的题目一定要分析二分的直线斜率和凸壳上斜率相同的情况！具体题目具体分析，假设 [l,r][l, r][l,r] 这一段斜率都相同，目标点在这个区间，你算出来的是 lll 还是 rrr，你二分是求 ≥n\ge n≥n 还是 ≤n\le n≤n 都要考虑清楚！

1.3 整体二分

1.3.1 算法介绍

考虑一类能够二分答案的问题，实际上可以将所有的询问一起二分：将所有询问放在分治中，判断分治中心是否大于等于其答案，并将询问分成两部分。因为每个询问在分治树的每一层中至多出现一次，所以复杂度是 O(Qlog⁡V)O(Q\log V)O(QlogV)，其中 VVV 是二分答案的值域。

乍一看和直接二分差不多，但是考虑一类问题：如果不能够快速算出一处点值怎么办？这个时候二分就失效了。但是如果维护的信息相邻处的点值可以快速计算的话，就可以利用分治中心移动的距离不大的性质，这就是整体二分。

整体二分常用于解决支持离线的第 kkk 大问题。

2. 分治系列

分而治之，分治要求维护信息的性质良好。

2.1 普通分治

2.1.1 算法介绍

最简单的分治可以做信息拆分与合并。通常要考虑跨越分治中心的信息合并。

2.1.2 例题

I. CF1442D Sum

给定 nnn 个不降的数组，有一个值 ansansans，初始为 000，你需要进行如下操作 kkk 次：选择一个数组，把 ansansans 加上数组的第一个元素，之后把他删除。求 ansansans 的最大值。

∑n≤106,n,k≤3000\sum n\le 10^6, n, k\le 3000∑n≤106,n,k≤3000

1.5s,500MB1.5s, 500MB1.5s,500MB

利用不降的性质，观察到最优解所有被选择的数组中，只有一个只选一部分，其他都选完了，否则一定可以通过替换使得答案更优。

那么枚举哪一个数组不是完整的，对于其他的数组，我们只用跑 01 背包就可以了，但是这样复杂度依然太高，你发现我们对于每一个数组都要对其他所有数组跑一遍背包，时间复杂度太劣了，能否重复利用信息呢，所以想到分治，递归左区间的时候把右区间加入背包，递归右区间的时候把左区间加入背包，时间复杂度优化至 O(nklog⁡n)O(nk\log n)O(nklogn)。

2.2 二维分治

2.2.1 算法介绍

多了一维，但是实际上没什么变化。交错地对每一维进行分割即可。

2.2.2 例题

时间复杂度 O(n2klog⁡n)O(n^2k\log n)O(n2klogn)。

2.3 CDQ 分治

2.3.1 算法介绍

如果要求计算分治中心左右两部分信息的合并，这两部分求值的先后顺序并没有要求。如果要求计算较早操作对较晚操作的影响，那么可以通过处理左子树的信息，来计算左子树对右子树的贡献。

其思想类似于操作分块，都是考虑操作的并是否能用简单的信息代替。

CDQ 分治常用于处理三维偏序问题，维护手段通常是一维排序，二维分治，三维数据结构维护。

2.4 线段树分治

2.4.1 算法介绍

按时间建立线段树，将操作覆盖到线段树上的点，那么处理一个时间的答案就是从叶子到根所有节点信息的并。利用可回退数据结构可以通过在线段树上 dfs 即可得到每个时间的答案。

注意，线段树可以处理强制在线的问题。

2.4.2 例题

I. P5787 二分图 /【模板】线段树分治

时间复杂度 O(mlog⁡mlog⁡n)O(m\log m\log n)O(mlogmlogn)。

IV. CF603E Pastoral Oddities

给定一张 nnn 个点的无向图，初始没有边，依次加入 mmm 条带权边，每次加入后询问是否存在一个边集，满足每个点度数均为偶数。若存在，还需要最小化边集中的最大边权。

首先有一个结论是，一个边集满足条件等价于每个联通块大小为偶数。显然如果存在一个联通块为奇数，那么这个联通块总的度数为奇数，矛盾。如果联通块大小为偶数也一定能够构造出来：求出一个生成树，然后从叶子开始向上匹配即可。

对于静态版本，把边排序做一遍最小生成树，用并查集维护一下奇联通块的个数即可。对于这道题，考虑线段树分治，如果一条边没有加入最优边集，那么他以后也不会再加入了。所以只需要从右往左分治，每次递归到叶子的时候找答案，在把边的影响区间加到线段树上即可。

时间复杂度 O(mlog⁡mlog⁡n)O(m\log m\log n)O(mlogmlogn)。

2.5 基于分治中心性质的写法

2.5.1 算法介绍

如果分治中遵循先递归左子树，再递归右子树的法则，那么维护一个指针去“跟踪”分治中心，这个指针移动的距离是 O(nlog⁡n)O(n\log n)O(nlogn) 的。利用这个性质可以获得优秀的做法。

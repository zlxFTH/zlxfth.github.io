串串小结

发表于 2022-12-19 更新于 2023-05-11 分类于 Notes

简易写一个串串小结，便于自己复习。

KMP

border

定义一个字符串 SSS 的 border 集合为满足如下两个条件的字符串 TTT 构成的集合：

  • S≠TS\ne TS=T。
  • TTT 是 SSS 的前缀，且 TTT 是 SSS 的后缀。

fail 数组

设 fail(i)fail(i)fail(i) 表示长度为 iii 的前缀的 border 集合中最长字符串的长度。

构建

考虑依次枚举 iii，维护目前的 border 长度 jjj，假设在 i−1i - 1i−1 时 border 长度为 jjj，并且 j+1j + 1j+1 并不是 iii 的 border 长度，不断跳 fail 函数，直到满足条件或者 j=0j = 0j=0，此时如果能够匹配则执行 j←j+1j\leftarrow j + 1j←j+1，于是求出了 iii 的 border 长度。

KMP 复杂度基于均摊，因为每个 iii 至多会让 jjj 加 111，所以时间复杂度为 O(n)O(n)O(n)。

字符串匹配

常用用法是进行单模式串和文本串匹配。

先求出模式串的 fail 数组，从前往后扫文本串，仿照构建函数，如果不匹配就不断跳 fail，如果当前匹配长度刚好是模式串长度则匹配上了，注意匹配上之后要跳一次 fail。

最小循环节

求出这个串的 border，如果满足：

∣S∣≡0(mod∣S∣−∣border(S)∣)|S| \equiv 0\pmod {|S| - |border(S)|} ∣S∣≡0(mod∣S∣−∣border(S)∣)

则这个串的最小循环节长度为 S−∣border(S)∣S - |border(S)|S−∣border(S)∣，否则长度为 ∣S∣|S|∣S∣。

考虑对应相等平移即可。

弱周期引理

若 p,qp, qp,q 为 sss 的周期，且 p+q≤np + q\le np+q≤n，那么 gcd⁡(p,q)\gcd(p, q)gcd(p,q) 也是它的周期。

Z 函数

在线性时间内求出一个模式串的所有后缀和文本串的 LCP。

注意第一个值需要暴力算。

ACAM

用于处理多模式串匹配问题。

就是 Trie 和 KMP 的结合体，Trie 树上每个节点维护一个 failfailfail 表示所有串中最长的后缀等于前缀的节点（类似 border，只是多个串一起）。

怎么建 fail？bfs 就可以了，一个建图优化是将空儿子直接指向其 fail 的对应儿子。

因为 fail 的原因，根设成 000，减少一些写法上的细节。

Manacher

回文中心：对于一个回文串，我们定义它的回文中心为，到字符串内到两端距离相等的位置（可以在两个字符中间）。

回文半径：对于一个回文串，我们定义它的回文半径为，其回文中心到两端之间的字符数。

Manacher 可以求出所有回文中心向两边延申的长度，时间复杂度 O(n)O(n)O(n)。

思路是维护最靠右已知的回文串，最右侧即为 mxmxmx，中心记作 ididid，如果当前 i≤mxi\le mxi≤mx，那么可以继承一些 ididid 为回文中心的长度，在这之后再暴力向外拓展即可。

因为回文串有奇偶之分难以处理，手动在两个字符之间插入一个分隔符，长度变成原来两倍然后再处理。

SA

构建

核心思想是求出两个有用的数组：sa，rk，分别表示排名为 iii 的后缀的位置和 iii 位置的排名。

对后缀排序的思想为倍增，利用计数排序可以做到 O(nlog⁡n)O(n\log n)O(nlogn)。

数组需要开 n+1n + 1n+1，保险起见直接 2n2n2n。

height 数组：ht[i]=lcp(sa[i],sa[i−1])ht[i] = lcp(sa[i], sa[i - 1])ht[i]=lcp(sa[i],sa[i−1])，根据结论 ht[rk[i]]≥ht[rk[i−1]]−1ht[rk[i]]\ge ht[rk[i - 1]] - 1ht[rk[i]]≥ht[rk[i−1]]−1，可以 O(n)O(n)O(n) 求出。

应用

  • 两个串的最长公共前缀

lcp(sa[i],sa[j])=min⁡(ht[i+1,⋯j])lcp(sa[i], sa[j]) = \min(ht[i + 1, \cdots j]) lcp(sa[i],sa[j])=min(ht[i+1,⋯j])

  • 比较一个字符串的两个子串的大小关系

直接转成比较后缀即可。

SAM

性质

SAM 自动机上是个 DAG，从根开始任意一条路径对应一个子串，走到终止状态对应一个前缀。

一条路径对应一个本质不同子串，一个本质不同子串对应一条路径。

一个串的所有出现的结束位置构成的集合称为 endpos，endpos 相同的串称为 endpos 等价类，每个节点相当于一个 endpos 等价类，所有走到这个节点的路径对应的子串属于同一个等价类。

endpos 集合里面的子串长度是连续的，设 uuu 节点的 endpos 集合内的子串的最长长度为 len(u)len(u)len(u)，并且其他的串都是最长串的后缀。

假设当前 endpos 对应的子串长度为 minlen(u)∼len(u)minlen(u)\sim len(u)minlen(u)∼len(u)，后缀链接 fa(u)fa(u)fa(u) 指向了 minlen(u)minlen(u)minlen(u) 的后缀且长度为 minlen(u)−1minlen(u) - 1minlen(u)−1 的串对应的 endpos 集合的节点，parent tree 是 u→fa(u)u\to fa(u)u→fa(u) 建立起来的树，可以发现从 uuu 沿 parent 树上走，所有 endpos 里面的串全是它的后缀并且长度刚好连续。

构建

SAM 采用增量建法，cur 和 last 都是终止状态，因为插入一个字符都相当于是新建了一个前缀。

在大概能想象它的流程的基础上背板子就行了。

空间记得开两倍。

字符集过大考虑用 map\text{map}map，不过复杂度会多 log⁡\loglog。

应用

  • 本质不同子串个数

每次新加一个字符的贡献是 len(cur)−len(fa(cur))len(cur) - len(fa(cur))len(cur)−len(fa(cur))，即当前节点对应 endpos 的大小。

  • 子串出现次数

每次新建的时候 cnt(cur)=1cnt(cur) = 1cnt(cur)=1，构建完成后 parent tree 上面 cntcntcnt 子树求和。

  • 原串一个子串的出现次数

先找到右端点 rrr 在 SAM 上对应的节点，倍增找到满足 len(u)≥r−l+1len(u)\ge r - l + 1len(u)≥r−l+1 的最上面的点，它的出现次数即是答案。

  • 优秀的拆分

求类似 AA 的结构的子串，枚举 AAA 的长度 lenlenlen，字符串上每隔 lenlenlen 放一个标记，那么每个长为 2×len2\times len2×len 的 AA 串显然恰好包含 222 个标记，问题转换成统计相邻两个标记的贡献，发现求出两个标记的 LCP 和 LCS 之后可以快速统计。

广义 SAM

还是很常用的。

给很多个串一起建 SAM，endpos 从一个串的结束位置变成了多个串的结束位置，便于处理多个串的子串信息。

一种伪广义 SAM 就是插入完一个串之后将 last 重新设为 111。

做法是先构建一个 Trie 树，维护队列按照 bfs 序的顺序加入节点，插入的时候不用新建节点，只需要取用之前 Trie 树上的节点即可。

clone 的时候要注意不能更新和 Trie 树上未加入的节点的儿子关系。

插入完之后返回 cur，继续把 cur 的儿子加入队列。

这样复杂度是关于 Trie 树节点数的线性，而不是伪广义自动机的关于所有串长度的线性。

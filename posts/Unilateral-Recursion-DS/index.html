单侧递归类数据结构

发表于 2023-02-01 更新于 2023-03-25 分类于 Notes

有一些数据结构，依靠只往一边递归来保证 log⁡\loglog 的复杂度，这类数据结构统称为单侧递归类数据结构，这里介绍几个常用的单侧递归类数据结构。

李超线段树

李超线段树是一种用于维护平面直角坐标系内线段关系的数据结构。最经典的应用是维护一个二维平面直角坐标系，支持动态插入线段，动态询问与直线 x=x0x = x_0x=x0​ 相交的已插入线段中交点 yyy 的最大值/最小值，即求当 x=x0x = x_0x=x0​ 时，max⁡{kix+bi}\max\{k_ix + b_i\}max{ki​x+bi​} 或 min⁡{kix+bi}\min\{k_ix + b_i \}min{ki​x+bi​}。

解决上文的问题，最暴力的做法是对于每一个询问的 x0x_0x0​ 暴力把所有线段全部扫一遍，单次询问复杂度 O(n)O(n)O(n)，这个似乎就是这个问题的瓶颈了，如何优化时间复杂度呢？发现单次询问的复杂度之所以很高，是因为遍历的集合大小过大，如果能排除不可能成为最优解的，那么就能降低时间复杂度，李超线段树就是利用了这个思想。

具体来说，李超线段树维护的是每个区间可能成为最优解的线段，这个线段 xxx 的取值范围必须完完全全包含这个区间。

维护区间内可能成为最优解的线段，且保证时间复杂度就成了这个算法的核心问题。对于这一问题，我们可以使用斜率和线段在当前区间中点上的取值，进行分类讨论。这里就以求最大值为例进行分析。

  1. 若当前区间没有线段，则直接将新的线段放入当前区间。

  2. 若当前区间 [l,r][l, r][l,r] 有旧线段，且旧线段的斜率小于新线段的斜率，假设当前区间的中点为 midmidmid：

  • 如果旧线段在 midmidmid 上面的值不小于新线段在 midmidmid 上面的值，那么区间 [l,mid][l, mid][l,mid] 内一定是旧线段最优，[mid+1,r][mid + 1, r][mid+1,r] 则不一定，将新线段递归进右儿子处理。

  • 如果旧线段在 midmidmid 上面的值小于新线段在 midmidmid 上面的值，那么区间 [mid+1,r][mid + 1, r][mid+1,r] 内一定是新线段最优，[l,mid][l, mid][l,mid] 则不一定，将当前区间的最优解换成新线段之后将旧线段递归进左儿子处理。

  1. 若当前区间 [l,r][l, r][l,r] 有旧线段，且旧线段的斜率大于新线段的斜率，假设当前区间的中点为 midmidmid：

  • 如果旧线段在 midmidmid 上面的值不小于新线段在 midmidmid 上面的值，那么区间 [mid+1,r][mid + 1, r][mid+1,r] 内一定是旧线段最优，[l,mid][l, mid][l,mid] 则不一定，将新线段递归进左儿子处理。

  • 如果旧线段 midmidmid 上面的值小于新线段在 midmidmid 上面的值，那么区间 [l,mid][l, mid][l,mid] 内一定是新线段最优，[mid+1,r][mid + 1, r][mid+1,r] 则不一定，将当前区间的最优解换成新线段之后将旧线段递归进右儿子处理。

  1. 若当前区间 [l,r][l, r][l,r] 有旧线段，且旧线段斜率等于新线段的斜率，比较截距 bbb，直接用截距大的替换截距小的。

上述分类讨论的框架可以直接放到线段树上面维护，单次查询只用遍历 O(log⁡n)O(\log n)O(logn) 个节点，所以单次查询时间复杂度为 O(log⁡n)O(\log n)O(logn)，一条线段最多分成 O(log⁡n)O(\log n)O(logn) 条线段，每条线段最多递归 O(log⁡n)O(\log n)O(logn) 层更新，所以单次修改的时间复杂度为 O(log⁡2n)O(\log^2 n)O(log2n)。

注意此类题目要特判线段垂直于 xxx 轴的情况。

合并上面的讨论，实际上可以写得更简洁，注意此时判断要加上等于号，因为会导致端点处相同的时候另外一条线段下放不下去，在一些要求取值相同编号最小的题目中会出错。

兔队线段树

为什么是这个名字？因为小粉兔有一篇《楼房重建》的题解。

兔队线段树是拿来维护前缀最大值的，即维护 (max⁡j<iaj)<ai(\max_{j < i} a_j) < a_i(maxj<i​aj​)<ai​ 的个数。

思路是每个区间维护这个区间的最大值、当前点是这个区间的前缀最大值的个数，然后每次 pushup 改成一个递归函数 calc(i,pre)\text{calc}(i, pre)calc(i,pre) 表示线段树上节点 iii 子树内在左边有 preprepre 的情况下的前缀最大值的个数。分情况讨论：

  • 如果 pre≤max_val(ls)pre\le max\_val(\text{ls})pre≤max_val(ls)，即前面的 preprepre 的值比左区间的最大值小，那么 preprepre 不会对右区间造成任何影响，沿用原来的答案，递归进左区间。

  • 如果 pre>max_val(ls)pre > max\_val(\text{ls})pre>max_val(ls)，即前面的 preprepre 的值比左区间的最大值大，那么左区间内没有任何值会有贡献，递归进右区间。

因为每次递归只会进入一边，最多递归 O(log⁡n)O(\log n)O(logn) 层，所以单次调用 calc()calc()calc() 的时间复杂度是 O(log⁡n)O(\log n)O(logn) 的，从上往下有 O(log⁡n)O(\log n)O(logn) 个节点需要 pushup，所以单次单点修改的时间复杂度为 O(log⁡2n)O(\log^2 n)O(log2n)。

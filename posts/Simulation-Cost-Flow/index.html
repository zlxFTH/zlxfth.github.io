模拟费用流

发表于 2023-05-04 更新于 2023-05-11 分类于 OI ， Notes

模拟费用流。

算法介绍

费用流的实质是反悔贪心，根据 EK 算法，每次是找到最短路增加 111 的流量，同时给反向边加同样的流量用于反悔。模拟费用流是利用某些费用流建图的特殊性质通过一些数据结构的手段来模拟费用流增广过程以达到优化时间复杂度的目的。

  1. 费用流有凸性，费用是关于流量的凸函数。
  2. 最小费用任意流：每次增广最短路，当单次费用为正时停止。
  3. 模拟费用流问题一般都从增量的角度考虑问题，分析添加一些点边之后对原图的费用流的影响，此刻只需要关注经过这些新的点边的增广路。
  4. 最常用的维护模拟费用流的数据结构是堆。

例题

P4694

  • kkk 个物品，先在 AAA 加工，再在 BBB 加工。nnn 天，每天 AAA 和 BBB 处各可以加工一个物品，代价为 ai,bia_i, b_iai​,bi​，一个物品可以在一天内加工完成。最小化费用。（1≤k≤n≤5×105,1≤ai,bi≤1091\le k\le n\le 5\times 10^5, 1\le a_i, b_i\le 10^91≤k≤n≤5×105,1≤ai​,bi​≤109）

模型：SSS 向 AAA 连流量为 111 费用为 aia_iai​ 的边，BBB 向 TTT 连流量为 111 费用为 bib_ibi​ 的边，i≤ji \le ji≤j 连边，跑费用流。可以费用流说明有凸性，wqs 二分消除 kkk 的限制，每对匹配加上 ttt 的代价。

模拟费用流的过程，假设后 i+1i + 1i+1 个位置的费用流已经流好了，现在加入 aia_iai​ 和 bib_ibi​ 这两个点，只用考虑新加入的 aia_iai​ 的带来的增广，用堆维护 bjb_jbj​，选出最小的一个进行匹配，如果总代价是正的则忽略，否则答案加上 ai+bj+ta_i + b_j + tai​+bj​+t 之后反悔，给反向边流 111 的流量相当于允许之后的 aka_kak​ 来替换选取当前的 bjb_jbj​，所以在堆里面加入 −ai−t-a_i - t−ai​−t。时间复杂度 O(nlog⁡2n)O(n\log^2 n)O(nlog2n)。

P1484

  • 给定一个序列 aia_iai​，从中选出 kkk 个数，要求选出的数不能相邻，求最大贡献。（1≤n≤5×1051\le n\le 5\times 10^51≤n≤5×105）

模型：以间隔建点，选第 iii 个点意味同时选第 i−1i - 1i−1 个间隔和第 iii 个间隔。SSS 向所有奇数间隔连流量为 111 的边，所有奇数间隔向相邻的偶数间隔连流量为 111 的边，费用为对应的 aaa，偶数间隔向 TTT 连边，限制流量 kkk 跑费用流。

考虑一次增广相当于是对“空【选空选……空选】空”进行翻转（左右两边一定是空，中间一定是空和选相间），将这样一个区间称作可选区间。

把每次增广将相邻的三个可选区间合并成一个，用链表和堆维护即可。

CF730I

  • 有 nnn 个人，每个人有两个权值 ai,bia_i, b_iai​,bi​，选出两个人的集合 A,BA, BA,B，∣A∣=p,∣B∣=s,A∩B=∅|A| = p, |B| = s, A\cap B = \varnothing∣A∣=p,∣B∣=s,A∩B=∅。AAA 集合的贡献为集合中人 aia_iai​ 的和，BBB 集合的贡献为集合中人 bib_ibi​ 的和。最大化 A,BA, BA,B 的贡献和。（2≤n≤3×103,p+s≤n,1≤ai,bi≤30002\le n\le 3\times 10^3, p+s \le n, 1\le a_i, b_i\le 30002≤n≤3×103,p+s≤n,1≤ai​,bi​≤3000）

模型：SSS 向 S1S_1S1​ 连流量为 ppp 的边，向 S2S_2S2​ 连流量为 sss 的边，S1S_1S1​ 和 S2S_2S2​ 分别向每个人连边，流量为 111，费用为对应贡献，每个人向 TTT 连流量为 111 的边，跑费用流。

有四种增广路：

  • S1→u→TS_1\to u\to TS1​→u→T：新增一个选 AAA 的。
  • S2→u→TS_2\to u\to TS2​→u→T：新增一个选 BBB 的。
  • S1→u→S2→v→TS_1\to u\to S_2\to v\to TS1​→u→S2​→v→T：将一个选 BBB 的改成选 AAA，新增一个选 BBB 的。
  • S2→u→S1→v→TS_2\to u\to S_1\to v\to TS2​→u→S1​→v→T：将一个选 AAA 的改成选 BBB，新增一个选 AAA 的。

用四个堆分别维护即可。

UOJ455

  • 一个人送一道菜，一共需要 nnn 道菜，第 iii 个人在 xix_ixi​，第 jjj 个餐厅在 yjy_jyj​，只能提供不超过 cjc_jcj​ 道菜，代价为 wjw_jwj​，第 iii 个人送第 jjj 个餐厅的菜的代价是 ∣xi−yj∣|x_i - y_j|∣xi​−yj​∣，最小化总代价。（2≤n,m≤1052\le n, m\le 10^52≤n,m≤105）

模型：离散化坐标，SSS 向每个人所在坐标连 (1,0)(1, 0)(1,0) 的边，每个餐厅向 TTT 连 (ci,wi)(c_i, w_i)(ci​,wi​) 的边，相邻坐标连 (∞,xi+1−xi)(\infin, x_{i + 1} - x_i)(∞,xi+1​−xi​) 的双向边。

首先 ∑ci<n\sum c_i < n∑ci​<n 无解。

在最左边虚拟一个代价无限大的餐厅，考虑按照坐标从左往右进行增广，用两个小根堆分别维护人和餐厅。

对于人：

  • 先在前面的餐厅中选一个最优的，贡献为 x+vx + vx+v。
  • 在人的堆里新增一个用于反悔 (−x−v)−x(-x - v) - x(−x−v)−x。

对于餐厅：

  • 如果匹配前面的人能够使得答案更少则匹配，贡献为 y+w+vy + w + vy+w+v。
  • 之后的人替换掉当前的人选择这个餐厅，在餐厅的堆中添加 −(y+w+v)+w-(y + w + v) + w−(y+w+v)+w。
  • 之后的餐厅匹配当前餐厅的人，在人的堆中添加 −y−w-y - w−y−w。

P6122「NEERC 2016」Mole Tunnels

给定一棵 nnn 个节点的树，第 iii 个点连向第 ⌊i/2⌋\lfloor i/2\rfloor⌊i/2⌋ 个点，边权为 111，第 iii 个点有一个可以容纳 cic_ici​ 只老鼠的洞，依次添加 mmm 只老鼠，第 iii 只老鼠住在 pip_ipi​ 号点上，对于每个 iii，求出让前 iii 只老鼠进洞的最小行动路径长度。

1≤n≤105,0≤ci≤m,1≤pi≤n1\le n\le 10^5, 0\le c_i\le m, 1\le p_i\le n1≤n≤105,0≤ci​≤m,1≤pi​≤n，保证一定可行。

模型：SSS 向每个老鼠连边 (1,0)(1, 0)(1,0)，每个洞向 TTT 连边 (ci,0)(c_i, 0)(ci​,0)，树上每条隧道连 (∞,1)(\infin, 1)(∞,1) 的双向边，跑费用流。

注意到树高只有 O(log⁡n)O(\log n)O(logn)。

每次增量添加一只老鼠，假设在 uuu 点，暴力跳到父亲 vvv 处，预先 DP 处 f(v)f(v)f(v) 表示 vvv 到子树内 ci≠0c_i\ne 0ci​=0 的点的最小费用，同时记录对应的点，那么可以 O(log⁡n)O(\log n)O(logn) 找出最短路径进行增广。

对于每个点 uuu 维护 g(u)g(u)g(u) 表示 uuu 和 fa(u)fa(u)fa(u) 之间的边的反向边流量状态，找出最优路径 u,vu, vu,v 之后可以暴力更新。

流量更新之后 fff 值变化的点为 uuu 到根的链和 vvv 到根的链的并，同样可以暴力维护。

时间复杂度 O(mlog⁡n)O(m\log n)O(mlogn)。

DP 及其优化

发表于 2023-01-11 更新于 2023-03-23 分类于 Notes

从学习与做题的角度来说，DP 主要分为状态设计和优化两部分。尤其要注意在设计算法前将题目进行一定的转化，将一个不熟悉的模型转化为我们更容易把握的模型。

1. 基本模型

1.1 背包问题

1.1.4 分组背包

同组物品只能选一个，对每一组进行一次 01 背包即可。

1.1.5 树上背包

把背包搬到了树上，大部分是对子树进行背包。

如果树上背包上下界都卡紧了复杂度会是 O(nm)O(nm)O(nm)，mmm 是背包大小。

1.2 区间 DP

1.2.1 算法介绍

如果题目中有“合并”的感觉，则很有可能是区间 DP。

对于一类配对最优化问题，往往证明其“包含不劣于相交”从而进行区间 DP。

区间 DP 只在区间中“最小值/最大值”处转移，这类套路叫做笛卡尔树 DP。

1.3 树形 DP

1.3.1 算法介绍

题目主要分为拓扑序问题和背包问题。

拓扑序问题：一般会设子树内拓扑序如何，然后合并子树时归并。

背包问题：前置 1.1.1.5，有时还需要换根 DP。

换根 DP：通常不会指定根节点，采用二次扫描的方法，第一次 dfs 预处理一些信息，第二次换根 DP。另外有一种换根 DP 的方式是 Up and Down DP，第一次“向上”处理子树内部的信息，第二次“向下”处理儿子的信息。

1.4 状压 DP

1.4.1 算法介绍

把一个状态压成一个数表示进行 DP，通常用 01 二进制表示。

1.4.1.1 常见位运算技巧

O(3n)O(3^n)O(3n) 枚举子集。

高维前缀和，O(n2n)O(n2^n)O(n2n) 求出所有集合的子集之和。

1.4.1.2 斯坦纳树

给一张带权无向图， nnn 个点，mmm 条边。有 kkk 个关键点，求取最小化代价的边，使得这 kkk 个关键点联通。

最后一定会选出来一个树状物。

设 f(i,s)f(i, s)f(i,s) 表示以 iii 为根包含了集合 sss 里面的关键点的最小代价。

从小往大枚举 sss，每次现在一个 iii 处跑一个子集枚举：f(i,s′)+f(i,s′⊕s)→f(i,s)f(i, s') + f(i, s'\oplus s)\to f(i, s)f(i,s′)+f(i,s′⊕s)→f(i,s)。

然后跑一个最短路 f(i,s)+dis(i,j)→f(j,s)f(i, s) + dis(i, j)\to f(j, s)f(i,s)+dis(i,j)→f(j,s)。

虽然做法看起来很蠢，可能会算重，但是只要保证最优解被我们统计到了就行了。

1.4.1.3 轮廓线 DP

给定一个 n×mn\times mn×m 带障碍的网格，从中选出若干个网格且相邻网格不能同时被选中，求方案数。

设 f(i,s)f(i, s)f(i,s) 表示转移到第 iii 行状态为 sss 的方案数。

转移枚举下一行的状态，复杂度太高。

不用一次枚举这么多东西，每一行从左往右依次枚举每一个是否选取就行了，相当于把一个“轮廓线”不停向右推，维护轮廓线上面的选取情况就行了。

时间复杂度 O(nm2m)O(nm2^m)O(nm2m)。

1.4.1.4 插头 DP

给出 n×mn\times mn×m 的方格，有些格子不能铺线，其他格子必须铺，形成一个闭合回路。问有多少种铺法？

n,m≤12n, m\le 12n,m≤12

1s,125MB1s, 125MB1s,125MB

（P5056【模板】插头 DP）

用于解决一类基于连通性状态的状态压缩动态规划。

本质上是状压 DP，采用轮廓线 DP 的方式。

插头 DP 主要在于插头的利用，一个格子有 444 个插头，一个存在的插头表示在它代表的方向上能与相邻的格子连接，不存在则表示不能。因为是哈密顿回路，一个格子恰好有 222 个插头，一个表示“进来”，一个表示“出去”，形成了 666 种插头状态：上下、左上、右上、左下、右下、左右。

轮廓线上从左往右的四个插头 a,b,c,da, b, c, da,b,c,d，如果 a,ca, ca,c 连通，b,db, db,d 一定不能连通。

于是联想到括号匹配，可以用括号表示法来表示状态。用 000 表示没有插头，111 表示左括号，222 表示右括号，记录轮廓线信息。

分类讨论折线处插头的状态转移：

  • 000000：虚线格内的插头肯定是右下，所以肯定是 121212。
  • 01/02/10/2001/02/10/2001/02/10/20：直走或者拐弯，可以很简单的讨论出来。
  • 11/22/2111/22/2111/22/21：对于 111111，把两个左括号并成一组，然后从当前位置往后面扫，找到匹配第二个左括号的右括号，再把右括号变成左括号，第二个左括号变成右括号，与第一个左括号匹配起来。对于 222222 同理，只不过是向前扫。212121 只需要把左右括号调换就可以了。
  • 121212：正常情况不能匹配，但是回路，在此结束。注意一开始要记录最后一个空格的位置。

用一个 Hash 表只把有用状态记录下来转移，可以节省复杂度。

时间复杂度大概是轮廓线状态数乘上格数，实际应用的时候往往通过搜索来确定实际状态数。

1.5 数位 DP

1.5.1 算法介绍

按位进行 DP，如果要考虑进位的话需要从小往大枚举，不然从大往小会更方便。

一些高维 DP 也借助了数位 DP 的思想，例如 NOIP 2021 T2。

比较方便的实现是记忆化搜索。

1.6 DP of DP

1.6.1 算法介绍

DP of DP 就是将一个内层 DP 的结果作为外层 DP 的状态进行 DP。

常用于计算一个 DP 答案为 kkk 的转移方案的数量。

通常而言，DP of DP 需要 encode 和 decode 操作，前者是将内层 DP 的结果用一定的方式把状态压起来，后者是在转移的时候将压缩的状态恢复方便转移。通常这两个部分需要预处理，进行外层 DP 的时候只需要压缩状态的转移，需要注意的是转移数和状态数大多数情况是不能脑测的，以实际搜索出来的结果为准。

DP 优化

DP 优化一般指的是优化时间复杂度，因为优化空间只有滚动数组一种方式。

DP 优化通常从优化状态和优化转移两方面入手。

优化状态一般需要利用问题的性质，具体情况具体分析。

优化转移通常有矩阵快速幂优化递推、数据结构加速转移以及利用决策单调性减少决策点。

矩阵快速幂优化 & DDP

矩阵快速幂通常用来优化一些转移层数很多，但是每一层状态很少的 DP。

把 DP 的状态写成一个矩阵的形式，把转移写成另外一个矩阵，一次转移就相当于乘上一个转移矩阵。

图上定长路径统计问题通常使用矩阵快速幂优化，现已套路化。

DDP：动态 DP，要求修改后快速求出 DP 结果，通常手段是维护一个线段树，每个节点维护一个矩阵，修改只需要从上往下改 O(log⁡n)O(\log n)O(logn) 个矩阵。如果是树，那么还需要修改 DP 的形式，用树剖解决（通常来说树上 DDP 用树剖足以解决，但是复杂度一般是两只 log⁡\loglog 的，利用全局平衡二叉树可以做到一只 log⁡\loglog）。

数据结构优化 DP

如果见到一些 DP 状态数大于实际复杂度情况，那么大概率就是用数据结构优化的。

如果转移点可以表示成一个前缀/后缀的形式，那么可以使用树状数组优化，最简单的应用就是最长上升子序列问题。

如果每一层之间的转移可以看成转换成简单的区间/单点修改和区间/单点查询，那么通常使用线段树来维护。

还有一些转移需要用到平衡树来优化。

决策单调性优化 DP

四边形不等式

区间包含单调性：如果对于任意 l≤l′≤r′≤rl\le l'\le r'\le rl≤l′≤r′≤r，均有 w(l′,r′)≤w(l,r)w(l', r')\le w(l, r)w(l′,r′)≤w(l,r) 成立，则称函数 www 对于区间包含关系具有单调性。

单调队列/单调栈优化

通过用于优化形如 f(i)=(max⁡i−k<j<if(j+bj)+vif(i) = (\max_{i - k < j < i} f(j + b_j) + v_if(i)=(maxi−k<j<i​f(j+bj​)+vi​ 的 DP 式子，当然求 min⁡\minmin 也是可以的。

应用的条件是求 max⁡\maxmax 的部分必须只和 jjj 有关。

维护一个单调递减的队列，显然如果队头的元素能用的话肯定使用队头的元素，否则队头在之后就没有用处了，直接弹出即可，更新完当前的 fif_ifi​ 之后需要加入队列中，如果不满足单调性，则一直弹出队尾，因为在之后队尾已经没有用处了。

每个元素只会入队和出队一次，因此时间复杂度为 O(n)O(n)O(n)。

如果 jjj 没有一个应用范围，在之后一直都能够使用，那么不需要弹出队头元素，那么这就是单调栈优化，本质上是一个道理。

一般形式 f(i)=A(i)B(j)+C(i)+D(j)f(i) = A(i)B(j) + C(i) + D(j)f(i)=A(i)B(j)+C(i)+D(j)，最大化/最小化 f(i)f(i)f(i)。

本质上是二分斜率，因为其在 DP 以外的一些题目中也有应用，所以这一部分放在《基础优化技巧》当中。

决策单调性分治

用于优化形如 f(i,j)=min⁡k≤jf(i−1,k)+w(k,j)f(i, j) = \min_{k\le j}f(i - 1, k) + w(k, j)f(i,j)=mink≤j​f(i−1,k)+w(k,j) 的 DP 式子。

如果 www 函数满足区间包含单调性和四边形不等式，则 fff 满足决策单调性，因此在相邻层之间转移只需要分治枚举决策点即可。

单层转移从 O(n2)O(n^2)O(n2) 优化到了 O(nlog⁡n)O(n\log n)O(nlogn)。

在实际应用中，大多数情况是通过打表/猜测 DP 式子的决策单调性而非证明。

二分队列

用于优化形如 f(i)=min⁡(f(j)+w(j+1,i))f(i) = \min(f(j) + w(j + 1, i))f(i)=min(f(j)+w(j+1,i))。

如果 www 函数满足区间包含单调性和四边形不等式，则 fff 满足决策单调性，可以使用二分队列来优化。

用单调队列维护决策点，相当于不断维护决策点的势力划分，每次通过二分找到划分点。

在有些题目中，如果两个决策点在一段区间内的贡献都是一样的，需要具体题目具体分析是划分给哪一个决策点，即代码中 find 函数里是写小于还是小于等于。

长链剖分优化 DP

算法介绍

长链剖分从一个节点到根的路径的轻边切换条数是 O(n)O(\sqrt n)O(n​) 级别的。

一般情况下可以使用长链剖分来优化的 DP 会有一维状态为深度维。

具体的，我们每个节点的状态直接继承其重儿子的节点状态，同时将轻儿子的 DP 状态暴力合并。

例题：CF1009F Dominant Indices

给定一棵以 111 为根，nnn 个节点的树。设 d(u,x)d(u,x)d(u,x) 为 uuu 子树中到 uuu 距离为 xxx 的节点数。

对于每个点，求一个最小的 kkk，使得 d(u,k)d(u,k)d(u,k) 最大。

1≤n≤1061\le n\le 10^61≤n≤106。

设 f(u,i)f(u, i)f(u,i) 表示以 uuu 为根的子树中距离 uuu 为 iii 的节点个数。

f(u,i)=∑f(v,i−1)f(u, i) = \sum f(v, i - 1) f(u,i)=∑f(v,i−1)

运用长链剖分优化，当前节点直接从重儿子继承，其余节点暴力合并。

相当于每次删去长度较小的链，每个点只会被暴力合并一次，所以时间复杂度和空间复杂度都是 O(n)O(n)O(n) 的。

实现上面需要动态开空间，利用指针方便地实现继承。
